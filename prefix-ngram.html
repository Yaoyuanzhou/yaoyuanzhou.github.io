§1 背景与动机

论文：Semantic IDs for Large-Scale Recommendation Systems

机构：Meta（Facebook 广告排序系统）

链接：arxiv.org/abs/2504.02137

状态：已在 Meta 广告排序系统生产部署超过 1 年

1.1 工业推荐系统中 ID 建模的三大困境

工业级推荐系统的核心是用 embedding 表示 item 的"身份"，传统方案给每个 item 分配随机哈希 ID，查询一张巨大的嵌入表。在 Meta 广告系统亿级物品规模下暴露出三个根本性缺陷：

困境 1：高基数性

广告数量极大，远超可实用的嵌入表大小。哈希压缩导致不同广告被迫共用同一个嵌入向量，产生严重的哈希碰撞，模型无法区分语义不同的广告。

困境 2：展示偏斜

流量分布极度不均衡：0.1% 的头部广告贡献了 25% 的展示量，大量尾部广告几乎没有历史数据，模型泛化能力极差。

困境 3：ID 漂移

约一半广告在 6 天内退出系统，同时大量新广告涌入。随机哈希 ID 无法在新老广告之间传递任何知识，新广告冷启动困难。

根本原因：随机哈希 ID 是孤立符号

两个语义相似广告（"耐克红色跑步鞋"和"耐克蓝色跑步鞋"）的 ID 在数值上毫无关联，模型无法从中学到任何可迁移知识。

1.2 核心思路：让语义相似的广告共享嵌入知识

解题思路：用内容模型把广告多模态内容压缩成向量，再用层次化量化把向量映射成一串离散 token（Semantic ID），使语义相似广告自然拥有相同的前缀 token，从而实现 embedding 参数共享。

核心 Insight：广告 A（"耐克运动鞋-红色"）和广告 B（"耐克运动鞋-蓝色"）经 RQ-VAE 量化后，前 2 层 code 完全相同（如都是 $c_1=47, c_2=132$），只在后几层不同。通过 Prefix-Ngram 展开，前缀 embedding 被两者共享——广告 A 的训练数据会直接改善广告 B 的表示，极大缓解长尾和冷启动问题。

2.1 内容嵌入提取

广告的语义内容来自多模态：文案标题、图片视觉、视频内容、落地页文本等。Meta 用专门的内容理解模型将这些信号融合为统一稠密嵌入向量 $\mathbf{e} \in \mathbb{R}^d$。

  • 内容嵌入是离线预计算的，不参与排序模型的在线推理
  • 内容模型独立训练，通常使用图文对比学习（CLIP 风格）或 LLM 文本表示
  • 广告更新时重新计算内容嵌入，排序模型在线推理时直接查 SID 对应嵌入表
💡 举例：内容嵌入的直觉含义

假设有三条广告：

  • 广告 A："Nike Air Max 跑步鞋 男款 红色"
  • 广告 B："Nike Air Max 跑步鞋 男款 蓝色"
  • 广告 C："女士高跟鞋 细跟 黑色"

经过内容理解模型，$\mathbf{e}_A$ 和 $\mathbf{e}_B$ 在空间中非常接近（余弦相似度 $\approx 0.95$），而 $\mathbf{e}_A$ 和 $\mathbf{e}_C$ 距离很远（余弦相似度 $\approx 0.3$）。这种语义近似性是 Prefix-Ngram 能共享 embedding 的基础。

2.2 RQ-VAE 残差量化：生成层次化 SID

RQ-VAE（Residual Quantized VAE）用 $L$ 层 codebook 逐步逼近输入向量，每层量化上一层的"残差"，最终用 $L$ 个离散 code 表示原始向量。

第 $l$ 层量化步骤：在 codebook $\{\mathbf{e}_k^{(l)}\}_{k=1}^{K}$ 中找最近邻：

$$c_l = \arg\min_{k} \|\mathbf{r}_{l-1} - \mathbf{e}_k^{(l)}\|_2^2, \qquad \mathbf{r}_l = \mathbf{r}_{l-1} - \mathbf{e}_{c_l}^{(l)}$$
符号说明
  • $\mathbf{r}_0 = \mathbf{e}$：初始残差为内容嵌入本身
  • $\mathbf{r}_l$：第 $l$ 层量化后的残差，代表"还未被捕获的信息"，模长随层数增加而减小
  • $K = 2048$：每层 codebook 大小；$L = 6$：总层数
  • $c_l \in \{1,\ldots,K\}$：第 $l$ 层分配到的 code 索引（整数）
  • $\mathbf{e}_{c_l}^{(l)}$：第 $l$ 层第 $c_l$ 个码向量，从 $\mathbf{r}_{l-1}$ 中减去后得到更细的残差

最终输出 SID $= (c_1, c_2, \ldots, c_6)$，一个由 6 个整数组成的层次化 code 序列。越靠前的层代表越粗的语义粒度（大类），越靠后越细（细节差异）。

💡 举例：RQ-VAE 逐层量化过程

以 $L=3$（简化）为例，假设内容嵌入 $\mathbf{e} = [0.8, 0.6, 0.3]$：

  • 第 1 层：在 codebook 1 中找最近邻，得 $c_1 = 47$（对应大类"运动鞋"）。码向量 $\mathbf{e}_{47}^{(1)} = [0.7, 0.5, 0.2]$，残差 $\mathbf{r}_1 = [0.1, 0.1, 0.1]$
  • 第 2 层：量化残差 $\mathbf{r}_1$，得 $c_2 = 132$（对应中类"耐克跑步鞋"），残差缩小至 $\mathbf{r}_2 \approx [0.02, 0.01, -0.01]$
  • 第 3 层：量化极小残差，得 $c_3 = 8$（对应细节"颜色差异"）

最终 SID = (47, 132, 8)。广告 B（蓝色版）可能得到 SID = (47, 132, 15)——前两层完全相同，只在第 3 层不同。正是这个层次性使得 Prefix-Ngram 能实现知识共享。

2.3 Prefix-Ngram 特征构造：层次化共享的关键

有了 SID $= (c_1, c_2, \ldots, c_L)$ 后，Prefix-Ngram 将其展开为所有前缀元组，每个前缀元组独立查嵌入表，最终求和（sum pooling）作为 item 表示：

$$\mathbf{v}_{\text{item}} = \sum_{l=1}^{L} \text{Lookup}_l\bigl((c_1, c_2, \ldots, c_l)\bigr)$$
符号说明
  • $L = 6$：使用 Prefix-5gram（最多到 5 元组），但论文测试了 1~5 gram 的不同配置
  • $\text{Lookup}_l(\cdot)$：第 $l$ 个嵌入表的查找，输入元组 $(c_1,\ldots,c_l)$，输出维度 $d$ 的嵌入向量
  • 总参数量 $H \approx O(50\text{M})$（通过哈希将高阶前缀压缩到合理大小）
  • 1-gram 嵌入表大小 $K = 2048$；2-gram 约 $4\text{M}$；3-gram 以上哈希到 $\sim 50\text{M}$
💡 举例：Prefix-Ngram 如何实现知识共享与冷启动

广告 A（耐克红色跑步鞋）：SID = $(47, 132, 8, 5, 21, 3)$

广告 B（耐克蓝色跑步鞋）：SID = $(47, 132, 15, 2, 8, 11)$

展开后查询：

  • 广告 A：查 $(47)$、$(47,132)$、$(47,132,8)$、$(47,132,8,5)$、$(47,132,8,5,21)$ 共 5 个嵌入，求和
  • 广告 B：查 $(47)$、$(47,132)$、$(47,132,15)$、$(47,132,15,2)$、$(47,132,15,2,8)$ 共 5 个嵌入，求和

共享部分：$(47)$ 对应的 1-gram 嵌入和 $(47,132)$ 对应的 2-gram 嵌入完全共享。广告 A 被点击时，梯度也流向这两个共享嵌入，间接帮助广告 B 提升表示质量——即使广告 B 当天没有任何展示。

冷启动解法：一个刚上线的新广告，只要内容与已有广告相似，就能立刻"借用"已有广告的训练成果，而不必从零开始学习。这是传统随机哈希 ID 完全做不到的。

2.4 FLOPs 正则化：防止 Codebook Collapse

RQ-VAE 训练中有一个经典问题：Codebook Collapse——大量输入被量化到少数几个 code，大多数 code 几乎从不被使用，实际利用的 codebook 容量远小于理论容量 $K$。

本文提出 FLOPs 正则化，核心思想是最大化 codebook 使用均匀性：

$$\mathcal{R}_{\text{FLOPs}}^{(l)} = \Bigl(\sum_{k=1}^{K} f_k^{(l)}\Bigr)^2 - \sum_{k=1}^{K} \bigl(f_k^{(l)}\bigr)^2$$
符号说明
  • $f_k^{(l)}$：第 $l$ 层 codebook 中 code $k$ 在当前 batch 内被分配到的频次
  • $\bigl(\sum f_k\bigr)^2$：总频次平方，为常数（与分配方式无关）
  • $\sum (f_k)^2$：各 code 频次的平方和——分配均匀时最小，collapse 时（极少数 code 频次极高）最大
  • 最大化 $\mathcal{R}$（等价于最小化 $-\mathcal{R}$）促使各 code 使用频次均匀，防止 collapse

总训练损失（RQ-VAE 部分）：

$$\mathcal{L}_{\text{RQ-VAE}} = \|\mathbf{e} - \hat{\mathbf{e}}\|_2^2 + \alpha \sum_{l=1}^{L}\|\mathbf{r}_{l-1} - \mathrm{sg}[\mathbf{e}_{c_l}^{(l)}]\|_2^2 - \beta \sum_{l=1}^{L} \mathcal{R}_{\text{FLOPs}}^{(l)}$$
符号说明
  • $\hat{\mathbf{e}} = \sum_{l=1}^L \mathbf{e}_{c_l}^{(l)}$：量化后的重建向量
  • $\mathrm{sg}[\cdot]$：stop-gradient，分离 codebook 更新与 encoder 更新
  • $\alpha, \beta$：超参数，控制 commitment loss 与 FLOPs 正则化的权重
  • 第三项前有负号：加入正则化等价于最小化负均匀性
💡 举例：FLOPs 正则化的数值对比

假设 $K=4$，batch 内 100 个样本，两种极端情况：

  • 极度 collapse（$f_1=97, f_2=2, f_3=1, f_4=0$）：
    $\sum f_k^2 = 9414$，$\mathcal{R} = 10000 - 9414 = 586$（极小，说明很不均匀）
  • 完全均匀（$f_1=f_2=f_3=f_4=25$）：
    $\sum f_k^2 = 2500$，$\mathcal{R} = 10000 - 2500 = 7500$（最大，分配最均匀）

最大化 $\mathcal{R}$ 就是让分配尽可能均匀，使所有 2048 个 code 都被充分利用，codebook 的完整容量得到使用，嵌入参数也能充分训练。

2.5 接入排序模型

  • 在排序模型特征层，将 Prefix-Ngram 嵌入 $\mathbf{v}_{\text{item}}$ 替代或与随机哈希 ID 嵌入拼接，再输入 MLP
  • 每个 Prefix-Ngram 元组对应独立嵌入表，在排序模型训练中端到端联合微调（RQ-VAE 部分冻结）
  • SID 一旦生成，在广告全生命周期内稳定，不随展示量变化

点击损失率随前缀深度增加单调下降，验证了层次化特征的有效性。本文最终选择 Prefix-5gram（即 $l = 1 \sim 5$）作为生产配置。

Meta 广告排序系统在线 A/B 测试结果（持续部署超过 1 年）：
  • 在线核心指标提升：+0.15%（统计显著）
  • A/A 预测方差降低：43%——说明 SID embedding 比随机哈希 ID 更稳定，预测结果的随机波动显著减小

方差降低 43% 的含义：在广告系统中，预测稳定性至关重要。传统随机哈希 ID 在 A/A 测试（两组流量完全相同，理论上指标应该完全一致）中表现出较大的随机波动，这是因为哈希碰撞和训练噪声。SID embedding 因为有语义先验，在等价流量上的预测更加一致，这不仅提高了实验可信度，也直接对应更稳定的广告收益。

核心贡献总结

✅ 主要亮点

  • FLOPs 正则化防 codebook collapse，是 RQ-VAE 工业应用的重要工程技巧
  • Prefix-Ngram 展开天然实现粗细粒度的层次化知识共享
  • serving cost 不增加（总参数量 $H \approx 50\text{M}$，与随机哈希 ID 持平）
  • 方差降低 43%，提升实验可信度，对广告系统工程价值很高
  • 生产部署 1 年+，稳定性有充分验证

⚠️ 局限与注意

  • 内容嵌入质量决定 SID 质量上限，内容模型需要独立维护
  • SID 生成是离线的，对于极其频繁更新的广告（分钟级），存在内容-SID 不同步问题
  • RQ-VAE 训练本身有成本，需要额外的离线流水线
  • 论文发表时主要是广告场景，内容推荐（视频、图文）需要额外验证
