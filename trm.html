TRM：用语义 Token 替代 Item ID

一句话总结

Item ID 本质上是孤立符号，难以共享知识、不利于大模型 scaling；而 Semantic Token 位于更稳定、可共享的语义空间，更适合大规模排序模型持续扩张。

原文：https://arxiv.org/abs/2601.22694

代码：未开源

核心贡献：提出 TRM 框架，从 协同感知表示学习、混合 tokenization、联合优化 三个层面系统解决"语义 token 替代 item ID 后老 item 性能下降"的问题，并在线上搜索场景验证效果。

背景与动机

推荐系统中的大型排序模型通常把每个 item 视为一个独立的 ID，再为它分配一个独立 embedding。这种方式简单直接，但一旦 item 集合快速演化，就会暴露出四个根本问题：

  1. 冷启动问题：新 item 没有足够点击和交互，embedding 训练不充分。
  2. 知识遗忘：旧 item 退役后，embedding 中承载的知识一起消失，无法迁移给新 item。
  3. 分布不稳定：item ID 集合不断变化，导致 sparse 表分布波动，dense 参数也更难稳定收敛。
  4. 扩展瓶颈：embedding table 随 item 数量线性膨胀，参数巨大，但知识共享能力很差。
论文的问题意识非常明确：不是简单证明"token 比 ID 好"，而是要回答：为什么 token 更适合 scaling？以及为什么已有 semantic token 方法对老 item 会掉点？

作者给出的理论直觉是：Semantic Token 构成一个更封闭、更稳定、更有结构的集合。根据 scaling law：

$$ L \propto N^{-\beta} $$
符号说明
  • $L$：模型在测试集上的损失值
  • $N$：模型参数量
  • $\beta$：scaling 指数，越大表示增加参数带来的收益越显著（曲线越陡）

含义：作者认为 token-based 表示具有更小的有效维度和更好的语义平滑性，因此能获得更大的 $\beta$，即同样增加参数时收益更大。

相关工作如 TIGER、OneRec、SemID 已尝试用 semantic token 替代 item ID，但它们在新 item 上常有优势，在老 item 上却容易掉点。TRM 的真正创新点在于把这个现象拆开讲清楚，并给出有针对性的解决方案。

方法详解

TRM（Token-based Recommendation Model）由三个模块组成：先学一个"带用户行为语义"的 item 表示，再把它 token 化（同时保住泛化和记忆），最后用判别式 + 生成式联合训练把 token 结构吃干榨尽。

模块 I：Collaborative-aware Representation（协同感知表示学习）

目标：学习一个既懂内容语义、又懂用户行为偏好的 item embedding，作为后续 tokenization 的基础。分两个阶段。

第一阶段：In-domain Captioning（领域内标注预训练）

用短视频领域数据微调多模态大模型（Qwen2.5-VL），让模型学会理解视频帧、标题、ASR 和 OCR，并生成符合该业务域的 caption。损失函数是标准自回归生成目标：

$$ \mathcal{L}_{cap} = -\mathbb{E}_{(V,T)} \sum_{k=1}^{|T|} \log P\bigl(t_k \mid t_1,\ldots,t_{k-1}, V;\,\theta\bigr) $$
符号说明
  • $V$：视频的视觉输入（帧图像等）
  • $T = t_1, \ldots, t_{|T|}$：目标 caption 的 token 序列，$|T|$ 是序列长度
  • $t_k$：caption 中第 $k$ 个 token
  • $t_1,\ldots,t_{k-1}$：生成 $t_k$ 时已知的前缀 token（自回归条件）
  • $\theta$：多模态大模型的参数
  • $\mathbb{E}_{(V,T)}$：对所有训练样本 $(V, T)$ 求期望（即对数据集取均值）

整体含义：对每个视频，用已知的前缀 token 预测下一个 token，最小化负对数似然。这是标准的语言模型训练目标。

第二阶段：Collaborative Alignment（协同对齐）

仅靠内容语义还不够——推荐系统真正关心的是"用户视角下什么内容相近"。所以第二阶段引入协同过滤信号，用对比学习把 embedding 朝用户行为空间拉齐：

$$ \mathcal{L}_{align} = -\mathbb{E}_{(a,b)\in\mathcal{P}} \log \frac{\exp\!\bigl(\mathrm{sim}(h_a,h_b)/\tau\bigr)}{\displaystyle\sum_{b'\in\mathcal{B}} \exp\!\bigl(\mathrm{sim}(h_a,h_{b'})/\tau\bigr)} $$
符号说明
  • $(a,b)$：一对正样本——要么是"query–item 正反馈对"，要么是"高协同相似度的 item–item 对"
  • $\mathcal{P}$：所有正样本对的集合
  • $h_a, h_b$：样本 $a$、$b$ 的 dense embedding（从 MLLM 最后一层 mean pooling 得到）
  • $\mathrm{sim}(\cdot,\cdot)$：余弦相似度
  • $\tau$：温度超参数，控制分布的"尖锐程度"，越小则正负样本区分越明显
  • $b'$：batch 内的其他样本（用作负样本）
  • $\mathcal{B}$：当前 mini-batch 中的所有样本集合（in-batch negatives）

整体含义：让正样本对的 embedding 尽量靠近，同时与 batch 内其他样本拉开距离。这是标准的 InfoNCE 对比学习损失。

两阶段的总表示学习目标：

$$ \mathcal{L}_{rep} = \mathcal{L}_{cap} + \lambda_{align} \cdot \mathcal{L}_{align} $$
符号说明
  • $\mathcal{L}_{cap}$：第一阶段的 caption 生成损失
  • $\mathcal{L}_{align}$：第二阶段的协同对齐对比损失
  • $\lambda_{align}$：平衡两个损失的超参数权重

直觉例子

视频 A 是"做蛋糕"，视频 B 是"做茶点"。从画面和文本看两者语义很近。

为什么要引入协同信号

如果 A 和 B 的受众完全不同，纯内容表示会把它们放太近；加入协同行为后，两者会被拉开，得到更符合推荐目标的 embedding。

模块 II：Hybrid Tokenization（混合 Token 化）

这是本文最核心的创新。已有 semantic token 方法对老 item 掉点的根因是：RQ-Kmeans 等残差量化只能粗粒度聚类，token 序列的简单聚合无法表达组合语义。

例如：token_3 表示"蛋糕"，token_16 表示"蜡烛"，$[3, 16]$ 的简单求和/拼接并不会天然表达"生日"这个语义概念——组合知识丢失了。

TRM 的解决方案：Gen-token + Mem-token 并行保留

  • Gen-tokens（泛化 token）：由 RQ-Kmeans 生成，共约 20,480 个，负责粗粒度语义共享，适合新 item 与长尾 item。
  • Mem-tokens（记忆 token）：用 BPE（Byte Pair Encoding）算法从高频 token 组合中生成，最多 20M 个，负责记住老 item 的高频组合模式。

二者通过 Wide & Deep 结构融合：

$$ h_{hybrid} = h_{deep}(\mathrm{gen\text{-}tokens}) + h_{wide}(\mathrm{mem\text{-}tokens}) $$
符号说明
  • $h_{hybrid}$：最终融合后的 item embedding，用于下游排序模型输入
  • $h_{deep}(\cdot)$：Deep 侧网络，以 gen-tokens 为输入，强调跨 item 的语义泛化
  • $h_{wide}(\cdot)$：Wide 侧（浅层记忆），以 mem-tokens 为输入，强调 item 粒度的组合记忆
  • $\mathrm{gen\text{-}tokens}$：粗粒度语义 token 序列（来自 RQ-Kmeans）
  • $\mathrm{mem\text{-}tokens}$：细粒度组合 token（来自 BPE 高频 k-gram 挖掘）

Deep 侧加了 random dropout 防止过拟合；Wide 侧不加 dropout，保持对高频组合的稳定记忆。

模块 III：Joint Optimization（判别式 + 生成式联合训练）

如果只把 token 当普通离散特征喂给排序模型，token 的层级结构没有被真正利用。TRM 加入生成式目标，让模型显式学习 token 序列内部的依赖关系。

判别式目标：预测用户行为（CTR / Like / Real-play 等）

$$ \mathcal{L}_d = \mathbb{E}_{(X_Q,\,X_I,\,X_U,\,Y)}\;\mathrm{BCE}\!\bigl(Y,\;P(\hat{Y} \mid X_Q, X_I, X_U;\,\theta_d)\bigr) $$
符号说明
  • $X_Q$：query 特征（用户搜索词等）
  • $X_I$：item 特征（多模态信息 + hybrid tokens）
  • $X_U$：user 特征（历史行为序列等）
  • $Y \in \{0,1\}$：用户对该 item 的实际行为标签（1 = 正反馈，0 = 无反馈）
  • $\hat{Y}$：模型预测的用户行为概率
  • $\theta_d$：判别式网络（RankMixer 等）的可学习参数
  • $\mathrm{BCE}(\cdot,\cdot)$：Binary Cross-Entropy 损失

生成式目标：对正样本 item 的 token 序列做 next-token prediction

$$ \mathcal{L}_g = \mathbb{E}_{(X_Q,\,X_U,\,Y)}\;\mathbf{1}[Y=1]\cdot\sum_{j=1}^{L}\mathrm{CE}\!\bigl(s_j,\;P(\hat{s}_j \mid X_Q, X_U, s_1,\ldots,s_{j-1};\,\theta_g)\bigr) $$
符号说明
  • $\mathbf{1}[Y=1]$：指示函数，只对正样本（用户发生正反馈的 item）计算生成损失
  • $s_j$：item 的第 $j$ 层 gen-token（ground truth）
  • $\hat{s}_j$：模型预测的第 $j$ 层 gen-token
  • $s_1,\ldots,s_{j-1}$：生成第 $j$ 个 token 时已知的前缀（自回归条件）
  • $L$：item gen-token 序列的总长度（本文用 5 层 RQ-Kmeans，所以 $L=5$）
  • $\theta_g$：生成式 Causal Transformer 的可学习参数（独立于 $\theta_d$）
  • $\mathrm{CE}(\cdot,\cdot)$：Cross-Entropy 损失

含义：给定 query 和 user 信息，让模型依次预测用户感兴趣 item 的每一层 token。token 有层级结构（前几层决定粗类，后几层决定细粒度），NTP loss 强制模型把这个结构学进去。

联合目标

$$ \mathcal{L} = \mathcal{L}_d + \lambda \cdot \mathcal{L}_g \qquad (\lambda = 0.1) $$
符号说明
  • $\mathcal{L}_d$：判别式损失（预测用户行为）
  • $\mathcal{L}_g$：生成式损失（预测 item token 序列）
  • $\lambda$：平衡两个目标的超参数，论文中设为 0.1（生成式目标只起辅助作用）

消融实验表明：把 $\lambda$ 设为 0 时 QAUC 从 +0.05% 降到 +0.01%，说明生成式目标带来的增益确实来自对 token 结构的利用，而非多加了参数。

个人理解

这篇论文最强的地方

  1. 把问题拆对了：不是停留在"token 好像更先进"的表面，而是具体指出已有 semantic token 方法为什么在老 item 上掉点。
  2. 方法设计有对应关系：Collaborative-aware 解决"内容语义 ≠ 用户语义"，Hybrid Tokenization 解决"泛化与记忆冲突"，Joint Optimization 解决"token 结构没被学到"。
  3. 理论和工程两头都站住了：既给出 scaling law 的理论解释，也做了线上 A/B，不是只在论文里成立。

还可以追问的点

  1. 链路较重：MLLM 预训练、RQ-Kmeans、BPE、联合训练，工程要求很高。
  2. 极新 item 收益有限：mem-token 对老 item 提升明显，但对刚上线的新 item 额外帮助不大。
  3. 生成目标只用正样本：负样本的 token 结构没有被系统利用，后续可能有提升空间。
核心 takeaway：Item ID 的问题在于把每个 item 当成孤立符号；Semantic Token 的优势在于把 item 放进了一个可共享、可组合、可扩展的语义空间。对大模型来说，后者显然更适合作为长期演进的基础设施。
